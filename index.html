Key Core Concepts
  • O(1) Constant time (independent of input size)
  • O(log n) Logarithmic time (binary search, BST heights)
  • O(n) Linear time (single loops, list scans)
  • O(n log n) Linearithmic time (heapsort, mergesort)
  • O(n²) Quadratic time (nested loops, bubble sort)

Complexity Reference
  • Selection Sort
    Worst/Best: O(n²) comparisons, O(n) swaps.
  • Insertion Sort
    Best: O(n) comparisons, worst: O(n²).
  • Bubble Sort
    Best: O(n) checks (sorted), worst: O(n²).
  • Binary Search
    Worst: O(log n), Best: O(1). Array must be sorted!

Gotchas & Rules
  • Always check if head == nullptr first!
  • When deleting the last node, update both head and tail references to nullptr.
  • In doubly-linked lists, remember to link node->next->prev = node (avoid Null Pointer Dereference!).

Array Queue: Circular Modulo

When writing a Queue using an array, we avoid element-shifting by using a circular array. The indexes wrap around using modulo:

rear = (rear + 1) % capacity;

This ensures O(1) push and pop!

Key BST Rules
  • Search Time: Worst: O(n) (unbalanced/chains), Average: O(log n).
  • In-Order Traversal: Recursively visits (Left, Node, Right). Prints BST elements in sorted order!
  • Pre-Order: (Node, Left, Right).
  • Post-Order: (Left, Right, Node).

Collision Resolution

When two keys hash to the same cell:

  • Linear Probing: Check cell (h(k)+i) % size.
  • Quadratic Probing: Check cell (h(k)+i²) % size.
  • Double Hashing: Check cell (h1(k) + i*h2(k)) % size.
  • Chaining: Each table index points to a linked list of collided nodes.

C++ Standard Template Library

STL Vectors, Lists, Sets, Maps, Stacks, and range-based loops

▶ Core STL Video (Aya Tell) ▶ Advanced STL Video (Iterators & Algos) 💻 STL.cpp source code
2
STL Container Reference Sheets
std::vector (Dynamic Array)

Constant-time tail inserts. Slow front inserts (requires shifting elements).

#include <vector>
vector<int> v;
v.push_back(10); // adds 10 to tail
v.pop_back();    // removes tail
int size = v.size();
int x = v[0];    // O(1) random access
std::set (Red-Black Tree)

A collection of sorted, unique keys. Implemented as a balanced BST.

#include <set>
set<string> s;
s.insert("Ahmad"); // O(log n)
s.erase("Ahmad");  // O(log n)
if (s.count("Ahmad")) { // checks presence (1/0)
    // found
}
std::map (Key-Value Balanced BST)

Associates unique keys to values. Node searches compare keys.

#include <map>
map<string, int> m;
m["Amman"] = 4000000;
m.erase("Amman"); // O(log n)
for (pair<string, int> p : m) {
    cout << p.first << " : " << p.second << endl;
}
3
Range-Based Loop Iteration

Iterating through std::set and std::map yields items sorted by key.

// Iterating over set:
for (int num : my_set) {
    cout << num << " ";
}

// Iterating over map:
for (pair<string, int> item : my_map) {
    cout << item.first << " has value " << item.second;
}
Complexity Reference
Class Insert Access
vector O(1)* O(1)
list O(1) O(n)
set O(log n) O(log n)
map O(log n) O(log n)
